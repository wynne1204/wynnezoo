# 对对碰规则文档

更新时间：2026-04-13

## 1. 基础设定

- 棋盘为 `3×3`，最多放置 `9` 个积木。
- 积木共有 `10` 种类型。
- 每次开盒获得的新积木，均从 `10` 种积木中独立、等概率随机产生。
- 任意时刻，棋盘上的积木总数都不超过 `9` 个。

## 2. 结算优先级

每次进入结算时，消除优先级固定为：

- `全家福 > 三连 > 对对碰`

同一个积木在同一次结算中：

- 只能参与 `1` 种消除。
- 如果同时满足多种消除条件，按上述优先级处理。

## 3. 全家福规则

当且仅当满足以下条件时，触发全家福：

- 当前棋盘上正好有 `9` 个积木。
- 这 `9` 个积木两两不同，全部为不同类型。

触发全家福后：

- 棋盘上的 `9` 个积木全部消除。
- 获得 `+5` 盲盒。

## 4. 三连规则

若未触发全家福，则继续判定三连。

三连定义为：

- 任意一条 `横向 / 纵向 / 斜向` 的长度为 `3` 的连线中，`3` 个积木类型相同。

三连结算规则：

- 每 `1` 组三连，获得 `+3` 盲盒。
- 参与三连的积木会被消除。
- 已参与三连的积木，不能再参与后续的对子结算。

## 5. 对对碰规则

在三连结算完成后，继续判定对子。

对子定义为：

- `2` 个相同类型的积木构成 `1` 对。

对子结算规则：

- 每 `1` 对，获得 `+1` 盲盒。
- 每对会消除其中 `2` 个积木。
- 若有 `3` 个相同积木，且它们未构成三连，则只算 `1` 对，消除 `2` 个，剩余 `1` 个保留。
- 一个积木不能同时参与多个对子。

## 6. 单轮结算流程

单轮结算按以下流程持续循环，过程中不补盒：

1. 先判定是否触发全家福。
2. 若未触发全家福，则判定三连。
3. 三连结算后，再判定对子。
4. 若本次消除后，盘面上仍存在可触发的全家福、三连或对子，则继续下一次判定。
5. 直到盘面上不存在任何可继续消除的组合时，本轮结算结束。

这里的“消到不能再消”定义为：

- 盘面上没有可以触发的全家福。
- 盘面上没有可以触发的三连。
- 盘面上没有可以触发的对子。

## 7. 补盒规则

- 单轮结算过程中不补盒。
- 只有当本轮所有可消除内容全部处理完后，才统一补盲盒。
- 补入数量为本轮结算累计获得的盲盒总数。
- 补盒后棋盘上的积木总数最多为 `9` 个。
- 如果可补盲盒数量超过当前空位数，也只能补到 `9` 个为止。

## 8. 清台奖励

清台定义为：

- 在某次结算过程中，棋盘上的积木被全部消除，盘面上没有任何积木。

清台奖励规则：

- 无论清台是由普通消除触发，还是由全家福触发，只要盘面最终为空，即判定为清台。
- 清台额外获得 `+5` 盲盒。

## 9. 单局结束条件

单局在以下条件下结束：

- 某次结算完成后，盘面上已无可继续消除的组合，且没有可补充的盲盒。

## 10. 判定示例

### 示例 1：全家福

- 当前棋盘有 `9` 个积木。
- `9` 个积木全部不同。
- 则直接触发全家福，`9` 个全部消除，获得 `+5` 盲盒。
- 若此时盘面为空，同时判定为清台，再额外获得 `+5` 盲盒。

### 示例 2：三个相同但不成三连

- 棋盘上有 `3` 个相同积木，但它们没有形成横向、纵向或斜向三连。
- 则不算三连。
- 在对子阶段，这 `3` 个积木只算 `1` 对。
- 消除其中 `2` 个，剩余 `1` 个保留。
- 本组获得 `+1` 盲盒。

### 示例 3：同一积木同时满足多个条件

- 如果某个积木既可能参与三连，也可能参与对子，
- 则该积木优先按三连结算。
- 结算后三连所使用的积木被消除，不再参与对子。
