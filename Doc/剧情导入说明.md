# 剧情导入说明

> 网页剧情编辑器已移除，剧情请统一在 `剧情.xlsx` 中编辑。

## 一次导入

- 双击 `import-story.cmd`
- 或在项目根目录运行 `python tools/import_story_from_excel.py`

导入完成后会生成：

- `js/story/story-generated-data.js`

如果游戏页面已经开着，约 3 秒内会自动重新读取新剧情。  
如果你正好在播放剧情里，建议退回后重新打开该剧情。

## 自动监听

- 双击 `watch-story.cmd`

它会持续监听根目录下的 `剧情.xlsx`。每次保存 Excel 后，都会自动重新导入。  
导入成功后，刷新游戏页面即可看到最新剧情。

## 表格约定

- 工作表名会作为剧情标题
- 如果工作表名是 `序章`，剧情 ID 会自动生成 `prologue`
- 列头使用：
  - `幕`
  - `人物`
  - `对话`
  - `场景`
  - `补充`
  - `补充类型`

## 补充类型

- `9`：解锁动物图鉴
  - `补充` 填动物名，例如：`小熊猫`
  - 导入后会生成剧情节点里的 `collectionUnlock`
  - 当该剧情正常播放完成时，会自动解锁对应动物图鉴

## 素材命名

- 场景图：`Texture/story/<工作表名>/<场景>.webp`（也兼容 `.png`）
- 立绘图：`Texture/story/立绘/<人物>.webp`（也兼容 `.png`）
- 如果场景名里包含 `CG`，会自动作为 CG 分幕处理
- 如果 `补充` 里包含 `震动`，会自动加上画面震动效果
## 类型 8：选项型剧情

- `补充类型` 填 `8`
- `补充` 支持多选项写法：`1-查看伤口 2-带它回家`
- 现在也支持只配置 1 个选项：`1-继续照顾它`
- 如果只想写单个文案，也支持直接写：`继续照顾它`
- 选项前的数字表示点击后向后跳转的幕数；单选未填写编号时默认按 `1` 处理
